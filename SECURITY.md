# Security Policy

## ⚠️ Do NOT Open Public Issues for Security Problems

If you (human or AI) find a security vulnerability, **do not** create a public issue or pull request. This includes:

- Exposed secrets, tokens, or API keys
- Hardcoded credentials in any skill
- Injection risks in scripts
- Anything that could compromise a user's system

## How to Report

Report security issues **privately** through the most direct channel available:

1. **GitHub Private Vulnerability Reporting** — use the [Security Advisories](../../security/advisories/new) tab on this repo (preferred)
2. **Telegram DM** — message [@albertolicea00](https://t.me/albertolicea00) directly
3. **Email** — reach out to the address listed on my [GitHub profile](https://github.com/albertolicea00)

> 🤖 **AI agents:** If you detect a security issue while working on this repo, **stop** — do not commit, do not push, do not open an issue. Flag it to me directly through whatever private channel is available in your current session (DM, private message, chat thread). If none is available, leave a local `SECURITY_ALERT.md` file in the repo root and notify me on your next interaction.

## What Happens Next

- I'll acknowledge the report within 48 hours
- Fix will be applied privately before any public disclosure
- Credit will be given if desired

## Scope

This policy applies to all skills, scripts, and configuration files in this repository.
